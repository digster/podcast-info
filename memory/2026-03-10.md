# 2026-03-10: Single-File Dashboard

## Summary
- Built `dashboard.html`, a standalone browser dashboard for exploring `joe_rogan_episodes.csv`
- Added client-side CSV parsing, derived fields, filters, charts, pagination, and an episode detail panel
- Added `tests/dashboard.spec.js` as a Playwright smoke test that opens the local HTML file and uploads the CSV
- Updated the README, architecture notes, prompt log, and ignore rules to reflect the new dashboard workflow

## Key Learnings
- A local single-file dashboard should use runtime CSV upload instead of `fetch` so it works reliably from `file://`
- The dataset needs a CSV parser that supports quoted multiline descriptions; a naive `split('\n')` parser is not sufficient
- The CSV currently leaves `show_name` empty for every row, so it is not a useful dashboard dimension yet

## Update: Table Search and Pagination

### Summary
- Moved the shared dashboard search control into the episode catalog toolbar and widened matching to a precomputed full-row search index
- Replaced fixed 20-row paging with selectable page sizes (`10`, `25`, `50`) plus compact numbered page navigation
- Expanded `tests/dashboard.spec.js` to verify search-by-date, page-size changes, direct page navigation, reset behavior, and drill-down still working together

### Key Learnings
- Precomputing a lowercased search index per row keeps shared search responsive without splitting the table into a second filtering system
- Large local datasets need windowed page-number rendering; showing every page button would overwhelm the single-file UI
